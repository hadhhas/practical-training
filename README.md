//项目简介


本项目是一个基于Web前端技术的艺术图库系统，是我的大二JavaScript实训课程作业。这是一个纯前端的应用，通过HTML、CSS和JavaScript实现了一个功能完整的艺术图片浏览平台。用户可以在系统中注册登录、浏览各种艺术图片、查看详细信息、收藏喜欢的作品，并在个人中心管理自己的资料和收藏。


项目的核心目标是综合运用所学的前端技术，包括DOM操作、事件处理、本地存储、响应式设计等，构建一个用户体验良好的艺术欣赏平台。所有数据都保存在浏览器本地（localStorage），因此不需要后端服务器支持，适合作为学习项目展示。


项目结构
项目采用清晰的文件结构组织，确保代码的可维护性和可读性：
//艺术图库
├── index.html          # 登录页面，用户从这里进入系统
├── register.html       # 注册页面，新用户可以创建账号
├── gallery.html        # 主图库页面，展示所有艺术图片
├── detail.html         # 图片详情页面，展示单张图片的详细信息
├── profile.html        # 个人中心页面，用户可以管理个人信息
├── css/                # 样式文件目录
│   ├── style.css      # 全局通用样式
│   ├── auth.css       # 登录注册页面的专用样式
│   ├── gallery.css    # 图库页面的布局和样式
│   └── profile.css    # 个人中心页面的样式
├── js/                 # JavaScript脚本目录
│   ├── auth.js        # 处理用户登录注册逻辑
│   ├── gallery.js     # 图库页面的交互功能
│   ├── detail.js      # 详情页面的功能实现
│   ├── profile.js     # 个人中心的功能实现
│   └── utils.js       # 通用的工具函数
├── images/             # 图片资源目录
│   ├── thumbnails/    # 缩略图，用于列表展示
│   └── full/          # 高清原图，用于详情页展示
└── README.md          # 项目说明文档



//功能列表


用户认证系统
用户注册：新用户可以通过表单注册账号
用户登录：已注册用户可以登录系统
表单验证：实时验证输入内容的有效性
记住登录状态：用户登录后保持会话状态


艺术图库浏览
瀑布流布局：图片以自适应的网格布局展示
响应式设计：适配各种屏幕尺寸的设备
图片懒加载：滚动到可视区域才加载图片
暗色模式：提供白天/夜间两种浏览模式


图片详情查看
高清大图展示：点击缩略图查看原图
作品信息展示：显示标题、作者、年代、描述等
收藏功能：可以收藏喜欢的作品
全屏浏览：支持全屏查看高清图片


个人中心管理
个人信息编辑：修改用户名、简介等信息
头像更换：上传并裁剪个人头像
收藏管理：查看和管理收藏的作品
浏览历史：记录最近查看的图片
账户设置：修改密码和安全设置


数据持久化
用户数据本地存储：使用localStorage保存用户信息
收藏记录保存：用户的收藏记录在本地持久化
浏览历史记录：自动记录用户的浏览行为
主题偏好保存：记住用户选择的主题模式



//运行步骤


环境准备
需要一个现代浏览器（推荐Chrome 90+、Firefox 88+、Edge 90+）
不需要安装任何额外的软件或环境


获取项目文件
解压到本地目录，确保所有文件都在同一个文件夹中


运行项目
方法一：直接双击index.html文件，用浏览器打开
方法二：使用VSCode的Live Server插件运行
方法三：使用Python的简单HTTP服务器运行
python -m http.server 8000
然后在浏览器中访问http://localhost:8000

//技术细节


前端技术栈
HTML5：使用语义化标签构建页面结构
CSS3：采用Flexbox和Grid实现布局，使用CSS变量管理主题
JavaScript ES6+：使用类、模块、箭头函数等现代语法
原生DOM API：不依赖任何框架，纯原生实现


核心功能实现
用户认证：在auth.js中实现，通过localStorage模拟用户数据库
图片懒加载：在gallery.js中使用IntersectionObserver API实现
瀑布流布局：在CSS中使用grid-auto-flow: dense实现自适应网格
头像上传：在profile.js中使用FileReader API处理图片文件
暗色模式：通过切换CSS自定义属性实现主题切换


数据管理方案
使用localStorage存储结构化数据
数据模型设计：
// 用户数据结构
const user = {
  id: 1,
  username: "用户名",
  password: "加密后的密码",
  avatar: "base64格式的头像",
  favorites: [1, 3, 5], // 收藏的作品ID数组
  history: [2, 4, 6],   // 浏览历史ID数组
  settings: {
    theme: "dark",      // 主题偏好
    language: "zh"      // 语言设置
  }
}


性能优化措施
图片懒加载：减少初始页面加载时间
事件委托：减少事件监听器数量
防抖节流：优化滚动和输入事件处理
本地缓存：减少重复数据加载


代码组织结构
模块化设计：每个功能模块对应一个JS文件
职责分离：HTML负责结构，CSS负责样式，JS负责行为
通用工具函数：提取公共方法到utils.js
错误处理：关键操作都有try-catch错误捕获


兼容性考虑
使用特性检测确保功能在旧浏览器中正常降级
添加CSS前缀确保样式兼容性
提供Polyfill支持（如果需要）


安全性考虑
前端输入验证：防止XSS攻击
密码本地加密：使用简单哈希函数处理
数据清理：防止恶意代码注入

这个项目涵盖了前端开发的核心技术，通过实际编码让我深入理解了DOM操作、事件处理、异步编程、本地存储等关键概念，为后续的前端学习打下了坚实基础。
